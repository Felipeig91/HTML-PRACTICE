GET, POST y Validación

En HTML, el atributo method del formulario decide cómo se envían los datos.

GET envía la información en la URL. Es útil para búsquedas o datos no sensibles.

POST envía la información en el cuerpo de la solicitud. Es ideal para enviar datos privados o grandes.

El atributo action define el archivo o ruta que recibe los datos. Si lo dejas vacío action="", el formulario se envía a la misma página.

La validación en HTML ocurre en el navegador con atributos como required, type="email", type="url" y pattern.

Validación con HTML

El navegador comprueba estos campos antes de enviar:

  • required: el campo no puede quedar vacío.
  • type="email": solo acepta formato de correo.
  • type="url": solo acepta una dirección web válida.
  • pattern="[0-9]{10}": valida un teléfono de 10 dígitos.
